# Specification: React + TypeScript Migration

## Summary

Migrate Context Modeler from vanilla JS + Vite to React 19 + TypeScript (strict mode) with a dark-first design system, shadcn/ui components, Framer Motion animations, and a backend-ready services layer. All 17 existing features carry forward 1:1.

## Context

The vanilla JS version is portfolio-ready (Phase 4D complete, 91 passing tests, WCAG 2.1 AA). However, the UI is visually flat -- no animations, no dark mode, cold color palette, poor typographic hierarchy. This migration elevates it to a portfolio centerpiece while making the codebase maintainable and extensible.

Design spec: `docs/superpowers/specs/2026-04-02-react-migration-design.md`
Design preview: `design-preview.html`

## Motivation

1. **Portfolio polish** -- The vanilla JS version works but doesn't impress. React + the dark-first navy/amber/teal design system creates a professional, modern feel.
2. **Maintainability** -- Vanilla JS is getting unwieldy. React's component model + TypeScript strict mode makes the codebase easier to extend and maintain.
3. Executive mandate.

## Risks

### 1. Scope creep
**Risk:** Migration balloons beyond 1:1 feature parity into new features or redesigns.
**Mitigation:** Strict adherence to the 17-feature list. New ideas go into a separate track.

### 2. Breaking the live site
**Risk:** GitHub Pages deployment breaks during transition.
**Mitigation:** All work on a feature branch. Vanilla JS `main` stays deployable until migration merges.

### 3. Data loss
**Risk:** Existing localStorage data from vanilla JS doesn't carry over to React.
**Mitigation:** Same `context-modeler:` key prefix and same data schema. StorageAdapter reads the same keys. Add migration test to verify round-trip.

## Technical Notes

- Full architecture and component tree defined in design spec

- shadcn/ui replaces hand-built modals, tabs, toasts, form inputs
- Framer Motion handles entrance animations, hover states, layout transitions
- Old `src/` vanilla JS will be archived to `src_vanilla/` for reference during migration
